## APEX IS AXIS-RELATIVE

`apex = argmax` over an ordering.
The ordering requires a quantity. "Apex" is undefined until you name the axis.

Candidate axes (a node can be apex on one, floor on another):

| axis | what it measures |
|---|---|
| reach | fan-out / broadcast amplitude |
| capital | resource flux controlled |
| status | attention gradient / deference owed |
| coercion | force monopoly |
| centrality | topological position (betweenness / eigenvector) |
| throughput | energy & matter actually moved ← substrate axis |
| epistemic | frame-generation accuracy (who's right) |
| constraint | boundary-condition authority (who sets what others optimize in) |

In `field_collapse.py`, `Φ_ext` couples to **one axis: REACH**.

The saddle-node crossing is driven by field amplitude on the order parameter.
The dynamics are blind to capital, status, truth, throughput.
A node drives collapse iff it maximizes fan-out — nothing else enters the equation.

∴ COLLAPSE_019 "apex amplification" was underspecified.
Correct reading: **"broadcast-reach-apex amplification."**
The lever the model identifies is the REACH channel, specifically.

## AXIS DECOUPLING — the real result hiding under the sloppy word

```
reach-apex  ≠  capital-apex  ≠  epistemic-apex  ≠  throughput-apex
```

And they can be **anti-correlated**:

- The collapse-driving node need not be rich, central, or right.
- Pure reach suffices. Capital that bought reach is upstream scenery;
  the REACH is what couples.
- An epistemically-floor node (false frame) with reach-apex drives
  the same collapse as a true one.

→ **"loud" is the coupling variable. Not "powerful," not "correct."**

## THREE READINGS OF APEX

### Scientific
`apex = argmax` of the coupling variable in the specific dynamics.
Axis matters absolutely — it IS the intervention lever.
Different model → different coupling channel → different apex.
Get the axis wrong and you suppress the wrong term (fear, capital, status)
while the actual driver (reach) runs untouched.
The axis isn't a labeling choice; it's the location of the lever.

### Narrative
Collapses ALL axes into one total order: "the powerful."
Single axis, single direction, single villain.
That collapse is not a description — it's the basin frame itself
(proximity-to-power as THE ordering). Narrative NEEDS one apex
because one apex = one story. It's the same single-axis flattening
caught in "poor = danger," run from the top instead of the bottom.
Narrative apex is a fiction of total order over a partial order.

### Substrate
There is no apex. Substrate has no intrinsic ranking — only
gradients, flows, throughputs, coupling geometries.
"Apex" is observer-imposed: pick a potential, pick a sign, call the
extremum "top." A high-fan-out node is just a high-coupling-degree
vertex — a geometric feature, not a height. "Apex" is like "up":
meaningless until you impose a field direction.

## APEX IS POSITIONAL, NOT INTRINSIC

```
collapse-driving capacity = broadcast/eigenvector centrality
  = property of POSITION in the coupling topology, not of the node.
```

Remove the node → the position can refill (whack-a-mole).

The substrate-correct lever is **topological**: change the coupling
structure that makes any position reach-apex — not delete the occupant.

This is why deplatforming one account barely moves the field:
you removed a node, not the position. The topology still has the slot.
